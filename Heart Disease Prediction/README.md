## 📌 Overview

Cardiovascular disease is one of the **leading causes of death worldwide**, and early risk detection can save lives. This project takes a real clinical risk model — built on well-established cardiac risk factors — and wraps it in a **clean, interactive web application** so that a risk assessment that would normally require a spreadsheet and a data scientist can be generated by anyone in seconds.

Enter a patient's basic vitals — age, cholesterol, blood pressure, smoking habits, heart rate, and glucose — and the app returns an instant, probability-scored prediction of heart disease risk.

## 🧠 What This Project Demonstrates

- **Full ML lifecycle ownership** — from model training to a shippable, containerized product
- **Clinical feature engineering** using established cardiovascular risk indicators
- **Class imbalance handling** with `imbalanced-learn` — critical for medical datasets where positive cases (disease) are typically the minority class
- **Feature scaling** via a persisted `scaler.pkl` to ensure inference-time consistency with training
- **Production-style project structure** — trained artifacts, app logic, and dependencies cleanly separated
- **Containerization with Docker** for reproducible, environment-agnostic deployment
- **User-centric UI/UX design** — sliders and dropdowns instead of raw form inputs, sensible clinical defaults, and clear visual risk communication (progress bar + color-coded verdict)

## 🩺 Input Features

The model evaluates seven key clinical and lifestyle indicators associated with cardiovascular risk:

| Feature | Description | Range |
|---|---|---|
| Sex | Biological sex of the patient | Male / Female |
| Age | Patient's age | 18 – 100 |
| Cigarettes Per Day | Smoking intensity | 0 – 60 |
| Total Cholesterol | Blood cholesterol level | 100 – 600 mg/dL |
| Systolic Blood Pressure | Upper BP reading | 80 – 300 mmHg |
| Heart Rate | Resting heart rate | 40 – 200 bpm |
| Glucose Level | Blood glucose | 40 – 400 mg/dL |

These are core risk factors drawn from long-standing cardiovascular epidemiology research (in the spirit of studies like the Framingham Heart Study), making the feature set clinically grounded rather than arbitrary.

## ⚙️ How It Works

```
User Input (Streamlit UI)
          │
          ▼
Feature Vector Assembly (7 clinical inputs)
          │
          ▼
StandardScaler.transform()  ──▶  scaler.pkl
          │
          ▼
Trained Classifier.predict() / predict_proba()  ──▶  heart_disease_model.pkl
          │
          ▼
Risk Verdict + Probability Score + Visual Feedback
```

1. **User Input** — Clinician or user enters patient vitals through an intuitive two-column form
2. **Preprocessing** — Inputs are scaled using the *exact* scaler fitted during training, preventing data leakage or inference mismatches
3. **Inference** — A pre-trained classifier (loaded from a serialized `.pkl` artifact) predicts both the class label and probability
4. **Result Rendering** — Streamlit displays a clear ✅ Low Risk / ⚠️ High Risk verdict, a probability metric, and a visual progress bar

## 🚀 Getting Started

### Option 1 — Run Locally

```bash
git clone <your-repo-url>
cd heart-disease-prediction

python -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate

pip install -r requirements.txt

streamlit run app.py
```

The app will be available at `http://localhost:8501`.

### Option 2 — Run with Docker

```bash
docker build -t heart-disease-app .
docker run -p 8501:8501 heart-disease-app
```

Then open `http://localhost:8501` in your browser.
